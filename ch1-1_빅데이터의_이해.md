# ch1 빅데이터 분석 기획

## ch1-1 빅데이터의 이해

### 01. 빅데이터 개요 및 활용

#### 	<1> 데이터 기초

##### [1] 데이터의 이해

###### 1. 데이터의 정의

- 데이터는 정보가 아니라 본질적으로 자료
- 의미를 담고 있으면 정보 / 의미를 담고 있지 않으면 데이터
- 데이터는 단순한 정보 객체로서의 가치뿐 아니라 다른 객체와의 상호관계 속에서 가치를 갖는다
  ex) 온도, 습도, 풍향, 기압 = 데이터  >> 일기예보
- 객관적 사실이라는 존재적 특성 + 추론 예측, 전망, 추정을 위한 근거로서의 당위적 특성

###### 2. 데이터의 유형

- 정량적 데이터(정형 데이터)
  - 수치로 표현할 수 있는 숫자, 도형, 기호 등의 데이터
  - 저장, 검색, 분석 활용에 용이
- 정성적 데이터(비정형 데이터)
  - 언어, 문자 등의 정형화되지 않은 데이터
  - 수치로 정형화하지 않은 데이터로 수집과 분석에 상대적으로 많은 비용이 듦

###### 3. 암묵지와 형식지

- 암묵지 (Tacit Knowledge)
  - 학습과 체험을 통해 개인에게 습득되어 있지만 겉으로 드러나지 않는 상태의 지식
  - 머릿속에 존재 / 언어나 문자를 통해 나타나지 않는 지식
  - 시행착오와 같은 경험을 통해 체득
- 형식지 (Explicit Knowledge)
  - 암묵지가 문서나 매뉴얼처럼 외부로 표출돼 여러 사람이 공유할 수 있는 지식
  - 교과서, 데이터베이스 등 어떤 형태로든 형상화된 지식
- 암묵지와 형식지의 상호작용
  - 암묵지가 형식지로 표출되고 연결되면서 지식이 형성 > 더 높은 가치 창출
  - 암묵지 상호작용
    - 공통화, 내면화
  - 형식지 상호작용
    - 표출화, 연결화

###### 4. 데이터와 정보

- 데이터(Data) : 개별 데이터 자체는 의미가 중요하지 않은 객관적 사실
  ex) A는 100원, B는 200원에 연필을 판매함
- 정보(Information) : 데이터의 가공, 처리 및 데이터 간 연관 관계 속에서 의미가 도출된 것
  단, 정보가 내포하는 의미가 항상 유용한 것은 아님
  ex) A연필이 더 저렴함
- 지식(Knowledge) : 데이터를 통해 도출된 정보를 구조화하여 유의미한 정보를 분류하고 개인적인 경험을 결합해 고유의 지식으로 내재화된 것
  ex) 더 저렴한 A로부터 연필을 사야함

- 지혜(Wisdom) : 지식의 축적과 아이디어가 결합된 창의적 산물
  ex) A의 다른 물건도 저렴할 것이라 추측

##### [2] 데이터베이스의 정의와 특징

###### 1. 데이터베이스 연혁

- 1950년대 미국에서 군수 물자를 관리하기 위해 데이터와 기지라는 합성어로 데이터베이스라는 용어가 탄생
- 우리나라느 1975년 미국의 CAC가 한국과학기술정보센터를 통해 데이터베이스를 처음 시작

###### 2. 데이터베이스의 일반적 특징

- 통합된 데이터 : 동일한 내용 중복 X\
- 저장된 데이터 : 컴퓨터로 접근 가능한 매체로 저장
- 공용 데이터 : 여러 사용자가 서로 다른 목적으로 데이터베이스 이용
- 변화되는 데이터 : 새로운 데이터 삽입, 기존 데이터 삭제, 갱신 > 정확한 데이터 상태 유지

###### 3. 데이터베이스 특징

- 정보의 축적 및 전달 측면
  - 기계가독성
  - 검색 가능성
  - 원격 조작정
- 정보이용 측면
- 정보관리 측면
- 정보기술발전 측면
- 경제 산업적 측면

##### [3] 데이터베이스 활용

###### 1. 기업 내부의 데이터베이스

- 인하우스 DB
- OLTP (Online Transaction Processing)
- OLAP (Online Analysis Processing)
- CRM, SCM

###### 2. 사회 기반 구조의 데이터베이스

- 물류 부문
  - CVO, EDI
  - CALS : 제품의 라이프 사이클 전반에 관련된 데이터 관리
  - PORT-MIS : 항만운영정보 시스템
  - KROIS : 철도운영정보 시스템
- 지리 부문
  - GIS
  - LBS : 위치정보 서비스
  - SIM : 공간정보 관리 시스템
- 교통 부문
  - ITS : 지능형 교통정보시스템
- 의료 부문
  - PACS
- 교육 부문



#### <2> 빅데이터의 이해와 가치

##### [1] 빅데이터의 이해

###### 1. 빅데이터의 정의

- 일반적 정의
  양적 개념 뿐 아니라 질적 개념도 포함하고 큰 용량과 복잡성으로 기존 어플리케이션이나 툴로 다루기 어려운 데이터셋의 집합

###### 2. 빅데이터의 특징

- 3V
  - Volume 크기
  - Variety 다양성
  - Velocity 속도
- 4V
  - Value 가치 or Veracity 정확성

###### 3. 빅데이터 출현 배경

- 산업계의 변화
  - 양질 전환의 법칙
  - 기업들이 보유 데이터 속에 숨어있는 가치를 발굴해 새로운 성장동력원으로 만들어냄
- 학계의 변화
  - 거대한 데이터를 다루는 학문 분야가 늘어나면서 필요한 기술 아키텍처 및 통계 도구 발전
- 관련 기술의 발전
  - 디지털화의 급진전, 저장기술 발전 및 가격 하락, 인터넷 모바일 시대, 클라우드 컴퓨팅

###### 4. 빅데이터에 거는 기대

- 산업혁명의 석탄, 철
  서비스 분야의 생산성을 발전시키고 혁명적 변화 야기
- 21세기 원유
  산업 전반의 생산성 향상 및 새로운 범주의 산업 발생
- 렌즈
  현미경의 렌즈가 생물학 발전에 끼쳤던 영향처럼 빅데이터도 렌즈처럼 산업 발전에 큰 영향
- 플랫폼
  공동 활용의 목적으로 구축된 유무형의 구조물

###### 5. 빅데이터가 만들어내는 변화

- 사전처리 > 사후처리
- 표본조사 > 전수조사
- 질 > 양
- 인과관계 > 상관관계



##### [2] 빅데이터의 가치와 영향

###### 1. 빅데이터의 가치

- 빅데이터를 통해 가치 있는 인사이트를 만들어내는 것이 중요
- 빅데이터 가치 산정의 어려움
  - 데이터 활용 방식
    특정 데이터를 누가, 언제, 어떻게 활용하는 지 알 수 없고 가치가 달라짐
  - 가치 창출 방식
    빅데이터는 기존에 없던 새로운 가치를 창출할 가능성이 있음
  - 분석 기술의 발전
    오늘은 가치 없는 데이터가 내일은 가치 있는 데이터가 될 수 있음

###### 2. 빅데이터의 영향

- 빅데이터가 가치를 만들어내는 5가지 방식
  - 투명성 제고로 연구개발 및 관리 효율성 제고
  - 시뮬레이션을 통한 수요 포착 및 주요 변수 탐색으로 경쟁력 강화
  - 고객 세분화 및 맞춤 서비스 제공
  - 알고리즘을 활용한 의사결정 보조
  - 비즈니스 모델과 제품, 서비스의 혁신
- 빅데이터의 영향
  - 기업
  - 정부
  - 개인
